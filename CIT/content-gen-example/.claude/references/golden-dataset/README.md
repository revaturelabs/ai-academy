# Golden Dataset — the reference outputs we test the pipeline against

This is the team's **regression reference**: complete, end-to-end artifact sets the current
pipeline produced and that passed every critic gate plus author review. When we change a skill,
a critic, or the config, we re-run these topics and compare against this set. When generation
goes outside expectations in day-to-day work, we add to this dataset (see "How to grow it").

**This is different from `../gold-examples/`.** Gold examples are the small curated set builders
*imitate* and critics *measure against* during generation (the taste bar). The golden dataset is
the *test fixture* — full topic chains used to detect regressions and document expected behavior.
Don't point builders here.

## Founding set (eval pass 3, 2026-06-11, branch `gen-improve`)

Four topics, one per week, all three depth tiers, all artifact types:

| Topic | Week | Depth | Why it's in the set |
|---|---|---|---|
| `1-8` Flowcharts | 1 | standard | Diagram-notation topic — diagram teaches the shapes themselves; pre-coding audience guard |
| `2-4` Pattern recognition | 2 | standard | ML scope-trap — week-3 mechanism must stay out; side-by-side contrast diagram |
| `3-6` Temperature & sampling | 3 | deep | Inversion test — week-1's deferred concept taught properly at its own depth; loop diagram |
| `4-3` RAG | 4 | deep | The config's deep exemplar — five-step pipeline diagram, week-4 defer discipline |

Each topic dir contains the full chain: `topic_corpus.md`, `artifacts/` (reading.md +
reading.meta.json, quiz.gift + quiz.meta.json, diagram.mmd + diagram.png), and
`critic-reports/` (the passing verdicts, including cycle counts where revision was needed).

Run conditions: `narrative_seed_only` mode (no external resources), critic loop ≤2 cycles.
Pass-3 results: corpora 2/4 first-shot → 4/4 cycle 2; diagrams, readings, quizzes all
first-pass. Known-good properties verified mechanically: zero assessment references, zero
helper-script artifacts, zero stem-level `#` in GIFT, diagrams present + vision-critic passed.

## How to use it

- **Before merging a pipeline change:** regenerate one or more golden topics in a sandbox and
  diff the behavior (critic verdicts, scope discipline, formatting shape, word bands) against
  this set. Verdict-level equivalence is the bar — prose will vary run to run; what must not
  vary is *passing*, staying on-headline, and the structural conventions.
- **As "what good looks like":** when reviewing at HITL #3, these show the expected shape for
  each depth tier and artifact type.

## How to grow it

When generation produces something **outside expectations** during real work:

1. Capture the offending artifact + which expectation it broke (one sentence).
2. If the fix is a skill/config change, land it, then regenerate the topic and add the
   *corrected* chain here as a new topic dir.
3. Add a `<topic>/NOTES.md` recording: what went wrong, the finding code, the fix commit.
   That note is the regression test's documentation — next time the behavior drifts, we know
   it's a recurrence, not a novelty.

Likewise, when a HITL #3 review approves an artifact the team considers exemplary, it can be
promoted here (and, if it represents *taste* rather than a test case, into `../gold-examples/`).

Provenance note: founding set generated by the pipeline and approved by the content author
(2026-06-11). Critic JSONs are kept verbatim — they are part of the fixture.
